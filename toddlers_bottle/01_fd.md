En arrivant sur le serveur, mon premier reflex est de faire un `la -la`
pour voir ce qui se cache ici.

```bash
fd@ubuntu:~$ ls -la
total 48
drwxr-x---   5 root fd      4096 Apr  1  2025 .
drwxr-xr-x 118 root root    4096 Jun  1  2025 ..
d---------   2 root root    4096 Jun 12  2014 .bash_history
-r-xr-sr-x   1 root fd_pwn 15148 Mar 26  2025 fd
-rw-r--r--   1 root root     452 Mar 26  2025 fd.c
-r--r-----   1 root fd_pwn    50 Apr  1  2025 flag
----------   1 root root     128 Oct 26  2016 .gdb_history
dr-xr-xr-x   2 root root    4096 Dec 19  2016 .irssi
drwxr-xr-x   2 root root    4096 Oct 23  2016 .pwntools-cache
```

On voit que le flag se trouve là. Et qu'on ne dispose probablement pas
des autorisations pour le lire. Preuve (bien qu'il n'en fallait pas vraiment):

```bash
fd@ubuntu:~$ cat flag
cat: flag: Permission denied
```

On voit aussi qu'on a du code source en c et le programme a été compilé.

À l'execution, on nous demande d'entrer un chiffre. On va donc en déduire que
le chiffre qu'on rentre devrait nous permettre d'avoir accès au flag.

```c
fd@ubuntu:~$ cat fd.c
#include <stdio.h>
#include <stdlib.h>
#include <string.h>
char buf[32];
int main(int argc, char* argv[], char* envp[]){
    if(argc<2){
        printf("pass argv[1] a number\n");
        return 0;
    int fd = atoi( argv[1] ) - 0x1234;
    int len = 0;
    len = read(fd, buf, 32);
    if(!strcmp("LETMEWIN\n", buf)){
        printf("good job :)\n");
        setregid(getegid(), getegid());
        system("/bin/cat flag");
        exit(0);
    }
    printf("learn about Linux file IO\n");
    return 0;
}
```

La lecture du code source nous apprend plusieurs choses :

* On déclare un buffer de 32 caractères
* On vérifie le nombre d'arguments passés à l'executable
* On convertit l'argument passé en integer puis on lui soustrait 0x1234 (4660)
* La fonction read() lit depuis fd dans le buffer

On est dans un challenge qui parle de `file descriptor` en Linux, on peut en
déduire que c'est à ca que correspond fd.

On a 3 file descriptors :

* 0 pour l'entrée standard
* 1 pour la sortie standard
* 2 pour la sortie d'erreur standard

Donc pour qu'on puisse passer la chaine `LETMEWIN` qui sera comparée dans la
structure conditionnelle, il faut que fd = 0 (pour qu'on lise dans stdin).

Il faut donc que `atoi( argv[1] - 0x1234 = 0`

Si on passe 4660 comme argument au script ./fd on devrait logiquement pouvoir
entrer la chaine `LETMEWIN` et obtenir le flag :

```bash
fd@ubuntu:~$ ./fd 4660
LETMEWIN
good job :)
```
